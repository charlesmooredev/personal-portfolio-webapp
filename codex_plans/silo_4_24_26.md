# Silo Records Console Redesign

## Visual Thesis

Black-glass records terminal with cyan phosphor text, dense dossier rows, thin wire dividers, and restrained system-boot motion.

## Content Plan

- Home: full-screen dossier database for Charles Moore and project records.
- Support: persistent terminal status/navigation controls for Projects, Skills, Contact, and external links.
- Detail: project, skills, and contact records open as fixed terminal panels.
- CTA: contact and external project links remain direct, readable, and keyboard accessible.

## Interaction Thesis

- Boot-in reveal for the console shell and header scan.
- Staggered project rows with selected-row sweep on hover/focus.
- Record panels open with compact terminal motion, with reduced-motion fallbacks.

## Development Notes

- Existing device shell lives primarily in `src/app/components/Desktop/index.tsx`.
- `MenuBar` contains macOS and iPhone status-bar affordances; it should be removed from the active shell.
- `Dock` and `AppIcon` create the strongest app-icon/dock metaphor; project access should become database rows instead.
- `AppWindow` currently uses traffic-light controls and rounded desktop window styling; it should become a terminal record overlay.
- Project data in `src/app/helpers/Projects.ts` is usable as-is.
- Rebuilt `Desktop` as a full-screen terminal workspace with a dossier database, operator panel, status header, external links, and desktop-only inspector.
- Replaced project app icons with dense record rows containing generated record IDs, roles, stack tags, and access/status labels.
- Reworked `AppWindow`, `ProjectWindow`, `SkillsWindow`, and `ContactWindow` into flat terminal records with command-style actions and no device/window chrome.
- Added CRT CSS tokens, scanlines, vignette/grid overlay, phosphor glow, selected-row sweep, cursor blink, custom terminal scrollbar, and reduced-motion fallbacks in `src/index.css`.
- Mobile now uses the same terminal layout in a stacked responsive flow, without any phone frame, dynamic island, home indicator, dock, or app icon grid.
- Fixed mobile scrolling by making the terminal shell a fixed `100svh` viewport and the main workspace the active scroll container.
